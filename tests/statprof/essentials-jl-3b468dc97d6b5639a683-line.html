43 """
44 @nospecialize
45
46 Applied to a function argument name, hints to the compiler that the method
47 should not be specialized for different types of that argument,
48 but instead to use precisely the declared type for each argument.
49 This is only a hint for avoiding excess code generation.
50 Can be applied to an argument within a formal argument list,
51 or in the function body.
52 When applied to an argument, the macro must wrap the entire argument expression.
53 When used in a function body, the macro must occur in statement position and
54 before any code.
55
56 When used without arguments, it applies to all arguments of the parent scope.
57 In local scope, this means all arguments of the containing function.
58 In global (top-level) scope, this means all methods subsequently defined in the current module.
59
60 Specialization can reset back to the default by using [`@specialize`](@ref).
61
62 ```julia
63 function example_function(@nospecialize x)
64 ...
65 end
66
67 function example_function(@nospecialize(x = 1), y)
68 ...
69 end
70
71 function example_function(x, y, z)
72 @nospecialize x y
73 ...
74 end
75
76 @nospecialize
77 f(y) = [x for x in y]
78 @specialize
79 ```
80 """
81 macro nospecialize(vars...)
82 if nfields(vars) === 1
83 # in argument position, need to fix `@nospecialize x=v` to `@nospecialize (kw x v)`
84 var = getfield(vars, 1)
85 if isa(var, Expr) && var.head === :(=)
86 var.head = :kw
87 end
88 end
89 return Expr(:meta, :nospecialize, vars...)
90 end

123
124 If `T` is an [`Integer`](@ref) type, an [`InexactError`](@ref) will be raised if `x`

126 range supported by `T`.
127
128 # Examples
129 ```jldoctest
130 julia> convert(Int, 3.0)

132
133 julia> convert(Int, 3.5)

135 Stacktrace:
136 [...]
137 ```

140 then it will return the closest value to `x` representable by `T`.
141

151

154 ```

157 `convert(T, x)` may alias all or part of `x`.
158 ```jldoctest
159 julia> x = Int[1, 2, 3];

161 julia> y = convert(Vector{Int}, x);

163 julia> y === x

476 """
477 @boundscheck(blk)
478
479 Annotates the expression `blk` as a bounds checking block, allowing it to be elided by [`@inbounds`](@ref).
480
481 !!! note
482 The function in which `@boundscheck` is written must be inlined into
483 its caller in order for `@inbounds` to have effect.
484
485 # Examples
486 ```jldoctest; filter = r"Stacktrace:(\\n \\[[0-9]+\\].*)*"
487 julia> @inline function g(A, i)
488 @boundscheck checkbounds(A, i)
489 return "accessing (\$A)[\$i]"
490 end;
491
492 julia> f1() = return g(1:2, -1);
493
494 julia> f2() = @inbounds return g(1:2, -1);
495
496 julia> f1()
497 ERROR: BoundsError: attempt to access 2-element UnitRange{Int64} at index [-1]
498 Stacktrace:
499 [1] throw_boundserror(::UnitRange{Int64}, ::Tuple{Int64}) at ./abstractarray.jl:455
500 [2] checkbounds at ./abstractarray.jl:420 [inlined]
501 [3] g at ./none:2 [inlined]
502 [4] f1() at ./none:1
503 [5] top-level scope
504
505 julia> f2()
506 "accessing (1:2)[-1]"
507 ```
508
509 !!! warning
510
511 The `@boundscheck` annotation allows you, as a library writer, to opt-in to
512 allowing *other code* to remove your bounds checks with [`@inbounds`](@ref).
513 As noted there, the caller must verifyusing information they can accessthat
514 their accesses are valid before using `@inbounds`. For indexing into your
515 [`AbstractArray`](@ref) subclasses, for example, this involves checking the
516 indices against its [`size`](@ref). Therefore, `@boundscheck` annotations
517 should only be added to a [`getindex`](@ref) or [`setindex!`](@ref)
518 implementation after you are certain its behavior is correct.
519 """
520 macro boundscheck(blk)
521 return Expr(:if, Expr(:boundscheck), esc(blk))
522 end

524 """
525 @inbounds(blk)
526
527 Eliminates array bounds checking within expressions.
528
529 In the example below the in-range check for referencing
530 element `i` of array `A` is skipped to improve performance.
531
532 ```julia
533 function sum(A::AbstractArray)
534 r = zero(eltype(A))
535 for i = 1:length(A)
536 @inbounds r += A[i]
537 end
538 return r
539 end
540 ```
541
542 !!! warning
543
544 Using `@inbounds` may return incorrect results/crashes/corruption
545 for out-of-bounds indices. The user is responsible for checking it manually.
546 Only use `@inbounds` when it is certain from the information locally available
547 that all accesses are in bounds.
548 """
549 macro inbounds(blk)
550 return Expr(:block,
551 Expr(:inbounds, true),
552 Expr(:local, Expr(:(=), :val, esc(blk))),
553 Expr(:inbounds, :pop),
554 :val)
555 end

698 """
699 invokelatest(f, args...; kwargs...)
700
701 Calls `f(args...; kwargs...)`, but guarantees that the most recent method of `f`
702 will be executed. This is useful in specialized circumstances,
703 e.g. long-running event loops or callback functions that may
